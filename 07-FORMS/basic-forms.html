Basic HTML Forms

What are HTML Forms?

HTML forms are used to collect user input and send it to a server for processing. Forms are essential for user interaction on websites, including login pages, contact forms, surveys, and e-commerce checkout processes.

Basic Form Structure

Every HTML form starts with the <form> element:

<form action="/submit" method="post">
    <!-- Form elements go here -->
</form>

Form Elements Explained

Form Element (<form>)

  • action: URL where form data is sent
  • method: HTTP method (GET or POST)
  • enctype: Encoding type for form data
  • target: Where to display response

Input Element (<input>)

  • type: Specifies input type (text, email, password, etc.)
  • name: Name for form submission
  • id: Unique identifier
  • value: Default value
  • placeholder: Hint text
  • required: Makes field mandatory

Label Element (<label>)

  • for: Associates label with input (matches input's id)
  • Improves accessibility
  • Clicking label focuses associated input

Textarea Element (<textarea>)

  • rows: Number of visible text lines
  • cols: Number of visible character widths
  • Used for multi-line text input

Select Element (<select>)

  • Creates dropdown menus
  • Contains <option> elements
  • multiple: Allows multiple selections
  • size: Number of visible options

Fieldset and Legend

  • <fieldset>: Groups related form elements
  • <legend>: Provides caption for fieldset
  • Improves form organization and accessibility

Form Attributes

Common Input Attributes:

  • required: Field must be filled
  • disabled: Disables the input
  • readonly: Makes input read-only
  • autofocus: Automatically focuses on page load
  • autocomplete: Controls browser autocomplete
  • pattern: Regular expression for validation
  • min/max: Minimum/maximum values
  • minlength/maxlength: Character length limits

Form Submission Methods

GET Method:

  • Data is sent in URL parameters
  • Visible in browser address bar
  • Limited data size
  • Good for search forms
  • Can be bookmarked

POST Method:

  • Data is sent in request body
  • Not visible in URL
  • No size limitations
  • Good for sensitive data
  • Cannot be bookmarked

Accessibility Best Practices

  1. Always use labels: Associate labels with form controls
  2. Provide clear instructions: Explain required fields and formats
  3. Use fieldsets: Group related form elements
  4. Error handling: Provide clear error messages
  5. Keyboard navigation: Ensure forms work with keyboard only
  6. Focus indicators: Make focus states visible
  7. Logical tab order: Ensure logical navigation flow

Form Validation

HTML5 provides built-in validation for many input types:

  • required: Field must be filled
  • pattern: Must match regular expression
  • min/max: Numeric range validation
  • minlength/maxlength: Text length validation
  • type-specific: Email, URL, number validation
